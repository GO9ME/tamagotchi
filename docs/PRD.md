# LifeGotchi — PRD (제품 요구사항)

한국형 **인생·커리어 다마고치** 웹앱. 아기 캐릭터를 만들어 밥·공부·운동·자기개발을
주기적으로 직접 눌러 키우고, 학생 → 취준생 → 직장인으로 성장하며 취업·업무평가·연봉협상·승진을
경험한다. 이 문서는 구현 기준이 되는 살아있는 스펙이며, 단계가 진행될 때마다 갱신한다.

---

## 1. 핵심 컨셉

- 단순 방치형이 아니라, **페이지를 켜두고 정해진 주기마다 직접 액션을 눌러야** 성장한다.
- 6대 차별점: ① 공부 30분 시작/완료 액션 ② 밥 안 주면 능력치 페널티 ③ 몸무게 관리
  ④ 전년도 자기개발 부족 페널티 ⑤ 취업 ⑥ 연봉협상.
- 모든 게임 공식은 `lib/game/` 순수 함수로 분리하고, 상태 수치는 0~100 clamp 한다.
- **효과 가시화 원칙**: 플레이어는 액션을 누르기 전에 어떤 컨디션/스탯이 얼마나 변하는지
  버튼에서 볼 수 있어야 하고(기본 효과 미리보기), 실행 후에는 대성공/부진 배수·과식 페널티까지
  합산된 **실제 적용 수치**를 토스트로 확인할 수 있어야 한다. (`lib/game/effectLabel.ts` 공용)

## 2. 비주얼 디자인 시스템 (확정)

전체 톤은 **2D 픽셀 다마고치**. "AI가 찍어낸" 느낌이 아니라 의도적으로 디자인된 레트로 감성.

### 2.1 폰트
- 본문/긴 글: **Pretendard** (가독성)
- 제목·라벨·숫자·버튼: 한국어 픽셀 폰트 **Galmuri** (`font-pixel`)
- 둘 다 jsdelivr CDN 링크로 `app/layout.tsx` `<head>` 에서 로드.

### 2.2 컬러 / 스킨
- 배경: 크림(`#FFF8F0`) + 잔잔한 도트 패턴.
- 잉크(외곽선/글자): `#2E2722` (UI), 마스코트 외곽선 `#3A2A22`.
- LCD 화면: 연한 크림-그린 `#E5EAD2` + 스캔라인.
- 카드/버튼: **하드보더**(3px) + 하드 픽셀 그림자, 플랫(블러·그라데이션 지양), 버튼은 눌림(translate) 효과.
- 이모지는 **8×8 픽셀 아이콘**(`lib/pixel/icons.ts`, `<PixelIcon/>`)으로 대체.

### 2.3 마스코트 (캐릭터)
- 스타일: **외곽선 + 흰 속칠**의 동글 몬스터(고전 다마고치 캐릭터 느낌). 통짜 실루엣 금지.
- 구현(`components/character/Mascot.tsx`): 16×16 실루엣에서 **외곽선을 자동 추출**하고 안쪽을
  흰색으로 채운다. 실루엣 문자열만 바꾸면 새 형태를 쉽게 추가할 수 있다.
- **표정 7종**: 기분최고/평온/시무룩/졸려요/배고파요/아파요/스트레스 (상태 수치로 자동 결정).
- **성장 단계별 진화**(`stageForm`): 단계가 오르면 생김새가 바뀐다.

  | 형태 | 적용 단계 | 느낌 |
  | --- | --- | --- |
  | `egg` | baby | 작은 알/방울 |
  | `chick` | child, elementary | 물방울 + 발 |
  | `round` | middle, high, university, jobseeker | 하트(귀) + 발 |
  | `tall` | employee, senior, retirement | 길쭉한 어른형 |

  > 확장 시 형태를 단계별로 더 세분화(고등/대학/직장인 각각 다른 몬스터)할 수 있다.

### 2.4 손안의 기기 (LCD)
- `components/character/TamaDevice.tsx`: **껍데기 색 = 캐릭터가 고른 색**(`MASCOT_COLORS`).
- 화면(LCD)에 마스코트 + 상태 핍(배고픔=하트, 체력=번개) 표시, 하단에 버튼 3개(장식).
- 앱 아이콘/파비콘/PWA 아이콘(PNG·SVG)도 동일 픽셀 마스코트에서 생성.

## 3. 데이터 / 개인정보 정책

- 회원가입 없음. 실명·전화·실제 회사/연봉 등 **개인정보를 수집하지 않는다.**
- 브라우저별 익명 ID(`guest_xxxx`) + 게임 진행 데이터만 저장.
- **localStorage**(zustand persist + migrate, 현재 version 19). 같은 브라우저에서 이어하기.
- 저장 레이어(`lib/storage/`)는 추상화되어 있어 추후 **Supabase Anonymous Auth + RLS**(기기 간
  이어하기, 서버 권위 계산)로 교체 가능. 랭킹엔 개인정보 미노출.

## 4.1 미니게임 / 행운 스탯 (확장 2 스펙)

돈이 아닌 **체력을 베팅**하는 운빨 콘텐츠. 쿨타임이 없어 원할 때 연속 플레이할 수 있고,
체력 경제(낮잠/수면 쿨타임)가 플레이 횟수를 자연스럽게 묶는다.

- **공통 규칙** (`lib/game/minigame.ts`)
  - 1회당 체력 **-12** (`MINIGAME_ENERGY_COST`), 쿨타임 없음, **6살부터**(`MINIGAME_MIN_AGE`,
    UI 숨김 + `canPlayMinigame` 로직 양쪽 게이트).
  - 꽝이어도 플레이마다 행운 **+0.2** (`LUCK_PER_PLAY`), 상한 **55** (`LUCK_CAP` — 소비처
    포화 지점, 도달 시 UI에 MAX 표시).
  - rand 는 0~1 주입(결정성) — 단위 테스트 가능.
- **행운 스탯** (`CharacterStats.luck`, 초기값 5)
  - 미니게임 당첨 확률 보너스: `luckBonus = clamp((luck-5)×0.3, 0, 15)` %p.
  - 일반 활동 대성공 확률 보너스: `greatChance += clamp((luck-5)×0.2, 0, 8)` %p.
- **슬롯머신 🎰**: 잭팟(기본 1.5%, +100만원 저축) / 대박(8%, +20만원) / 소소한 당첨(25%,
  +5만원) / 꽝(기분 -2·스트레스 +3). 기대값은 판당 +4만원 안팎으로 연봉 대비 무시할 수준.
- **행운 뽑기 🎁**: 희귀 캡슐(기본 4%, **스탯 포인트 +1**) / 스탯 캡슐(18%, 배분 가능 스탯 중
  랜덤 +1) / 일반 캡슐(기분·자신감·집중 중 하나 소폭 회복).

### 상점 뽑기 (확장 2 스펙)

옷/방 인테리어/자동차는 직접 구매 대신 **확률 뽑기**로 획득한다 (`lib/game/shopGacha.ts`).

- **공통 규칙**
  - 티켓값 고정: 옷 5만원 · 인테리어 150만원 · 자동차 3,000만원 (`GACHA_CONFIG`).
    티켓값은 풀 평균 정가보다 싸게 — 전부 모으면 직접 구매 합계보다 이득.
  - 풀 = **미소장** 아이템만(중복 없음 → 뽑다 보면 전부 모인다). 옷은 나이 조건(minAge)
    충족분만, 자동차는 **보유 티어보다 높은 차만** 등장(운 좋으면 티어 건너뛰기).
  - 레어 = 풀 상위 1/3 가격대. 레어 확률 `18% + 행운 보너스(최대 +15)`. 레어 당첨 시
    OutcomeBurst 연출, 마지막 남은 아이템(확정 지급)은 연출 없음.
  - 뽑을 때마다 행운 +0.2(미니게임과 동일, 상한 55). 정가 > 티켓값이면 토스트에 "이득!" 표시.
  - 획득 효과는 기존 구매와 동일(기분/자신감/행복도, 옷은 즉시 착용, 가구는 방에 렌더).
- 주거(이사)는 뽑기 대상이 아니다 — 대출 구조가 있어 직접 선택 유지.

### 자신감 관리 (확장 2)

- 자신감도 시간에 따라 **-0.8/시간**으로 감쇠(`DECAY_PER_HOUR.confidence`, 다른 감쇠보다 완만)
  — 한번 올리면 끝인 죽은 스탯이 아니라 관리 대상.
- 상시 회복 수단: **칭찬하기** 💖(자신감 +6·기분 +5·스트레스 -2, 쿨타임 1h), 근력 운동,
  옷 쇼핑, 면접 연습, 미니게임 당첨 등.
- 자신감은 면접 점수(`employment.ts`, 가중치 0.09)에 반영되므로 취준 시기에 특히 중요.

## 5. 구현 메모

- 게임 공식: `lib/game/`(constants, status, actions, study, weight, growth, engine, review, exam,
  gating, character, outcome, minigame, effectLabel 등).
- 상태: `lib/store/useGameStore.ts`(zustand + persist). `tick()` 에서 `applyDecay → runDueReviews`.
- 단계별 액션 잠금은 UI와 store 양쪽에서 강제(클라 우회 방지).
- 빌드 기준: `npm run build` 통과 유지, Vercel 배포 가능 상태.
